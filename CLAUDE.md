## What this repo is

`claude-assistant` is the orchestration layer for an autonomous research
assistant running on a local machine. It:

- Watches `Claude/Research/Requests/` in an Obsidian vault for new notes
- Spawns a Claude Code subprocess per request (parallel instances allowed)
- Each instance has MCP access to the Obsidian vault and a browser (Playwright)
- Writes research output to `Claude/Research/<topic>.md`
- On success, stamps the request note's frontmatter and moves it to `Claude/Research/Requests/Done/`
- On failure, stamps the request note and moves it to `Claude/Research/Requests/Error/`; writes a detailed error note to `Claude/Research/Errors/<topic>.md`

## Architecture

```
Obsidian vault
  Claude/Research/Requests/        ← drop .md request notes here
  Claude/Research/Requests/Done/   ← completed requests (status: done)
  Claude/Research/Requests/Error/  ← failed requests (status: error)
  Claude/Research/<topic>.md       ← Claude Code writes output here
  Claude/Research/Errors/<topic>.md← detailed error log on failure

watcher.py
  startup scan → skip notes already marked status: done or status: error
  watchdog Observer  →  2s debounce  →  asyncio.create_subprocess_exec
                                         ↳ claude --mcp-config mcp_config.json \
                                                  --allowedTools "mcp__markdown-rag__*,mcp__playwright__*" \
                                                  -p "<file content>"

mcp_server.py  (MCP stdio server used by each Claude Code subprocess)
  search_notes   — semantic search via RAG HTTP API
  read_note / list_notes / create_note / update_note / delete_note / lint_note
                 — vault file operations via obsidian-mcp-guard
```

### Request note lifecycle

- **Exit 0** — frontmatter stamped with `status: done`, `completed: <ISO date>`, `output: [[<topic>]]`; note moved to `Requests/Done/`
- **Non-zero** — frontmatter stamped with `status: error`; note moved to `Requests/Error/`; detailed error note (exit code, stdout, stderr) written to `Claude/Research/Errors/<topic>.md`

Existing frontmatter fields on the request note are preserved through both paths.

## Conventions

- Python 3.11+ — use `pathlib.Path` throughout, not `os.path`
- Shell scripts: POSIX `#!/bin/sh`
- `config.yaml` and `mcp_config.json` are in `.gitignore` — never commit them
- Logs go in `logs/` (also gitignored)
- Do not modify `mcp_server.py` or `tests/` without good reason — they are stable
